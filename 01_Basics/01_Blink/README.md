# 01 - Blink (Hello World)

## Περιγραφή
Το κλασικό πρώτο πρόγραμμα Arduino. Αναβοσβήνει το ενσωματωμένο LED της πλακέτας κάθε 500ms.

## Πού χρησιμοποιείται σε υλοποίηση IoT;
*   **Heartbeat Συστημάτων:** Ένα LED που αναβοσβήνει σε IoT Gateways δείχνει ότι η συσκευή λειτουργεί και δεν έχει "κολλήσει".
*   **Ένδειξη Σύνδεσης:** Σε έξυπνες πρίζες, το γρήγορο αναβόσβημα συχνά δηλώνει λειτουργία σύζευξης (pairing mode) με το δίκτυο.
*   **Status Indicators:** Σε αισθητήρες χωρίς οθόνη, το LED επιβεβαιώνει τη λήψη μέτρησης.

## Μαθησιακοί Στόχοι
- Εκμάθηση της δομής ενός Arduino sketch (`setup()` και `loop()`)
- Κατανόηση των βασικών εντολών: `pinMode()`, `digitalWrite()`, `delay()`
- Επαλήθευση σωστής εγκατάστασης IDE, board και port

## Η Βασική Δομή ενός Sketch

Κάθε πρόγραμμα Arduino (sketch) χρειάζεται δύο βασικές συναρτήσεις: `setup()` και `loop()`.

- `void setup()`: Ο κώδικας μέσα σε αυτή τη συνάρτηση εκτελείται **μία μόνο φορά**, όταν το Arduino ενεργοποιείται ή γίνεται επανεκκίνηση. Εδώ βάζουμε κώδικα αρχικοποίησης, όπως το να πούμε στο Arduino ποιες ακίδες θα χρησιμοποιήσουμε.

- `void loop()`: Ο κώδικας εδώ μέσα εκτελείται **συνεχώς, σε επανάληψη**, για όσο το Arduino είναι ενεργοποιημένο. Αυτή είναι η καρδιά του προγράμματός μας.

## Νέες Έννοιες

- `pinMode()`: Μια συνάρτηση που χρησιμοποιούμε μέσα στο `setup()` για να ορίσουμε τον ρόλο μιας ακίδας (pin). Μπορεί να είναι `INPUT` (για να διαβάζει δεδομένα) ή `OUTPUT` (για να στέλνει σήματα, π.χ., να ανάβει ένα LED).
- `digitalWrite()`: Στέλνει μια ψηφιακή τιμή σε μια ακίδα. Οι τιμές είναι `HIGH` (υψηλή τάση, δηλαδή "αναμμένο") ή `LOW` (χαμηλή τάση, δηλαδή "σβηστό").
- `delay()`: Σταματάει την εκτέλεση του προγράμματος για ένα συγκεκριμένο χρονικό διάστημα, το οποίο ορίζεται σε χιλιοστά του δευτερολέπτου (milliseconds). Για παράδειγμα, το `delay(1000);` είναι παύση για 1 δευτερόλεπτο.
- `LED_BUILTIN`: Μια ειδική σταθερά του Arduino που αντιστοιχεί στον αριθμό της ακίδας όπου είναι συνδεδεμένο το ενσωματωμένο LED. Στις περισσότερες πλακέτες (όπως το Uno), είναι η ακίδα 13. Χρησιμοποιώντας το `LED_BUILTIN` ο κώδικάς μας γίνεται πιο ευανάγνωστος και συμβατός με διαφορετικές πλακέτες.

## Υλικά που Χρειάζονται
- 1× Arduino Uno (ή συμβατή πλακέτα)
- 1× Καλώδιο USB

**Σημείωση:** Δεν απαιτείται εξωτερική καλωδίωση!

## Οδηγίες Σύνδεσης

1. Συνδέστε το Arduino στον υπολογιστή με USB
2. Δεν χρειάζεται breadboard ή εξωτερικά εξαρτήματα
3. Το `LED_BUILTIN` (pin 13) έχει ενσωματωμένο LED και αντίσταση

## Πώς να το Τρέξετε

1. Ανοίξτε το `blink.ino` στο Arduino IDE
2. Επιλέξτε **Tools → Board → Arduino Uno**
3. Επιλέξτε **Tools → Port → COM#** (το σωστό port σας)
4. Πατήστε το κουμπί **Upload** (→)
5. Παρατηρήστε το LED στο pin 13 να αναβοσβήνει

## Επεξήγηση Κώδικα

```cpp
void setup() {
  pinMode(LED_BUILTIN, OUTPUT);  // Ρύθμιση pin 13 ως έξοδος
}

void loop() {
  digitalWrite(LED_BUILTIN, HIGH);  // Άναμμα LED
  delay(500);                       // Αναμονή 500ms
  digitalWrite(LED_BUILTIN, LOW);   // Σβήσιμο LED
  delay(500);                       // Αναμονή 500ms
}
```

1.  `setup()`: Λέμε στο Arduino: "Θέλω να χρησιμοποιήσω το ενσωματωμένο LED για να στέλνω σήματα εξόδου".
2.  `loop()`:
    -   Ανάβει το LED.
    -   Περιμένει μισό δευτερόλεπτο.
    -   Σβήνει το LED.
    -   Περιμένει μισό δευτερόλεπτο.
3.  Αυτός ο κύκλος επαναλαμβάνεται συνεχώς, με αποτέλεσμα το LED να αναβοσβήνει.

## Πειραματισμός

Δοκιμάστε να αλλάξετε:
- Τις τιμές `delay(500)` σε `100` ή `1000` για διαφορετική ταχύτητα
- Διαφορετικές καθυστερήσεις για ON και OFF: `delay(200)` και `delay(800)`

## Τι συμβαίνει μέσα στο Arduino

Ο κώδικας που αναπτύσσεται στο Arduino IDE και μεταφορτώνεται στην πλακέτα ταξινομείται ως **firmware**. 

Το firmware είναι ένας ειδικός τύπος λογισμικού (software) που είναι ενσωματωμένος απευθείας σε ένα κομμάτι υλικού (hardware), συνήθως σε μνήμη non-volatile (που δεν χάνεται όταν διακοπεί η τροφοδοσία, όπως η flash μνήμη του Arduino). 

Ουσιαστικά, παρέχει τις βασικές οδηγίες για το πώς πρέπει να λειτουργεί το συγκεκριμένο υλικό και πώς να αλληλεπιδρά με άλλα εξαρτήματα (αισθητήρες, κινητήρες κ.λπ.)

Στην περίπτωση του Arduino:
- Το πρόγραμμά σας (sketch) μεταγλωττίζεται (compile) σε γλώσσα μηχανής και "ενσωματώνεται" στη μνήμη flash του μικροελεγκτή του (π.χ.  ATmega328P)
- Αυτό το Firmware εκτελείται απευθείας από τον επεξεργαστή ή όπως το λέμε "απευθείας στο μέταλλο" (bare metal), δηλαδή χωρίς κάποιο ενδιάμεσο λειτουργικό σύστημα (όπως τα Windows ή το Linux).
- To Firmware διευθύνει άμεσα τη συμπεριφορά του υλικού, όπως το πότε θα αναβοσβήσει ένα LED ή πώς θα διαβάσει έναν αισθητήρα.

Αξίζει να σημειωθεί ότι η ίδια η πλακέτα Arduino περιέχει επίσης ένα μικρό προ-εγκατεστημένο firmware, τον bootloader ("φορτωτής"), ο οποίος επιτρέπει τη μεταφόρτωση του δικού σας κώδικα (firmware εφαρμογής) μέσω USB.
