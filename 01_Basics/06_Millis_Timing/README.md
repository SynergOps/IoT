# 06_Millis_Timing - Μη-αποκλειστικός Χρονισμός

## Περιγραφή
Μάθετε πώς να κάνετε πολλά πράγματα ταυτόχρονα (Multitasking) χρησιμοποιώντας την `millis()` αντί για την `delay()`.

## Πού χρησιμοποιείται σε υλοποίηση IoT;
*   **Multitasking:** Ένας έξυπνος θερμοστάτης πρέπει να διαβάζει τη θερμοκρασία, να ενημερώνει την οθόνη και να ελέγχει το Wi-Fi ταυτόχρονα.
*   **Περιοδική Αποστολή:** Αισθητήρες περιβάλλοντος στέλνουν μετρήσεις στο Cloud κάθε 10 λεπτά, χωρίς να σταματούν να παρακολουθούν για κρίσιμα συμβάντα.
*   **Watchdog Timers:** Μηχανισμοί που επανεκκινούν τη συσκευή αν "κολλήσει" για συγκεκριμένο χρονικό διάστημα.

## Μαθησιακοί Στόχοι
- Κατανόηση της `millis()` για χρονισμό
- Μη-αποκλειστικός προγραμματισμός (non-blocking code)
- Διαχείριση πολλαπλών χρονιστών ταυτόχρονα
- Αποφυγή προβλημάτων του `delay()`

## Γιατί Millis(); Τα Προβλήματα του delay()

### ❌ Με delay():
```cpp
digitalWrite(LED, HIGH);
delay(1000);  // ⚠️ Μπλοκάρει όλο το πρόγραμμα για 1 δευτερόλεπτο
digitalWrite(LED, LOW);
delay(1000);  // ⚠️ Αν πατήσουμε κουμπί εδώ, το Arduino δεν θα το καταλάβει!
```

### ✅ Με millis():
```cpp
if (millis() - lastTime >= 1000) {
  // Εκτέλεση μόνο όταν περάσουν 1000ms
  // Στο ενδιάμεσο, ο κώδικας τρέχει ελεύθερα και μπορεί να διαβάσει το κουμπί!
}
```

## Πώς να το Τρέξετε

1. Ανοίξτε το `millis_blink.ino`
2. Ανεβάστε στο Arduino
3. Το LED αναβοσβήνει αργά (κάθε 1 δευτερόλεπτο)
4. **Κρατήστε πατημένο το κουμπί:** Το LED αναβοσβήνει γρήγορα (κάθε 100ms)
5. **Αφήστε το κουμπί:** Το LED επιστρέφει στον αργό ρυθμό

## Επεξήγηση Κώδικα

```cpp
const int LED_PIN = LED_BUILTIN;
const int BTN_PIN = 2;

unsigned long previousMillis = 0;
unsigned long interval = 1000; // Αρχική ταχύτητα

void loop() {
    // 1. Έλεγχος Κουμπιού (Άμεση Αντίδραση)
    if (digitalRead(BTN_PIN) == LOW) {
        interval = 100; // Γρήγορα όσο πατάμε
    } else {
        interval = 1000; // Αργά όταν αφήνουμε
    }

    // 2. Διαχείριση Χρόνου LED (Blink)
    unsigned long currentMillis = millis();
    if (currentMillis - previousMillis >= interval) {
        previousMillis = currentMillis;
        // ... Toggle LED ...
    }
}
```

### Βασικές Έννοιες:
- **millis()**: Επιστρέφει χρόνο σε ms από την εκκίνηση
- **unsigned long**: Τύπος για μεγάλους αριθμούς (μέχρι ~49 ημέρες)
- **Έλεγχος διαφοράς**: `now - last >= period` αποφεύγει rollover issues

## Σύγκριση delay() vs millis()

| Χαρακτηριστικό | delay() | millis() |
|----------------|---------|----------|
| Μπλοκάρει πρόγραμμα | ✅ Ναι | ❌ Όχι |
| Πολλαπλά timers | ❌ Δύσκολο | ✅ Εύκολο |
| Responsive input | ❌ Όχι | ✅ Ναι |

## Προσοχή: Rollover του millis()

### Τι είναι το Rollover;

Το `millis()` επιστρέφει έναν αριθμό τύπου `unsigned long`, ο οποίος έχει μέγιστη τιμή:

```
Μέγιστη τιμή unsigned long: 4,294,967,295 (2^32 - 1)
```

Αυτό σε milliseconds αντιστοιχεί σε:
```
4,294,967,295 ms ÷ 1000 ÷ 60 ÷ 60 ÷ 24 = ~49.7 ημέρες
```

**Μετά από 49 ημέρες, το `millis()` επιστρέφει στο 0 και ξεκινά από την αρχή!**

```
Χρόνος: ...  4,294,967,293  →  4,294,967,294  →  4,294,967,295  →  0  →  1  →  2  ...
          ms:                                                      🔄 ROLLOVER
```

---

### ❌ Λάθος Τρόπος (Δεν Λειτουργεί με Rollover)

```cpp
unsigned long lastTime = 4294967200;  // Κοντά στο όριο
const int interval = 500;

void loop() {
  unsigned long now = millis();
  
  if (now >= lastTime + interval) {  // ❌ ΠΡΟΒΛΗΜΑ!
    // ...
  }
}
```

**Τι συμβαίνει:**
```
Χρόνος                now      lastTime + interval    Συνθήκη
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
Πριν rollover:
T = 4294967200ms     4294967200    4294967200+500=4294967700    ❌ 4294967200 >= 4294967700? FALSE (σωστό)
T = 4294967300ms     4294967300    4294967700                    ✅ 4294967300 >= 4294967700? FALSE (λάθος!)
T = 4294967400ms     4294967400    4294967700                    ✅ 4294967400 >= 4294967700? FALSE (λάθος!)

Μετά rollover (⚠️ PROBLEM):
T = 0ms              0             4294967700                    ❌ 0 >= 4294967700? FALSE (λάθος!)
T = 100ms            100           4294967700                    ❌ 100 >= 4294967700? FALSE (λάθος!)
T = 1000ms           1000          4294967700                    ❌ 1000 >= 4294967700? FALSE (λάθος!)
```

**Αποτέλεσμα:** Ο χρονιστής **ποτέ δεν ενεργοποιείται** μετά το rollover! 🔴

---

### ✅ Σωστός Τρόπος (Λειτουργεί με Rollover)

```cpp
unsigned long lastTime = 4294967200;  // Κοντά στο όριο
const int interval = 500;

void loop() {
  unsigned long now = millis();
  
  if (now - lastTime >= interval) {  // ✅ ΣΩΣΤΟ!
    lastTime = now;
    // Εκτέλεση κώδικα
  }
}
```

**Τι συμβαίνει:**

Η αφαίρεση `unsigned long` έχει ειδικές ιδιότητες. Δουλεύει σωστά ακόμα και με overflow:

```
Χρόνος                now      lastTime    now - lastTime    Συνθήκη        Κατάσταση
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
Πριν rollover:
T = 4294967200ms     4294967200  4294967200    0              0 >= 500? ❌ FALSE    Περιμένει
T = 4294967300ms     4294967300  4294967200    100            100 >= 500? ❌ FALSE  Περιμένει
T = 4294967400ms     4294967400  4294967200    200            200 >= 500? ❌ FALSE  Περιμένει
T = 4294967600ms     4294967600  4294967200    400            400 >= 500? ❌ FALSE  Περιμένει
T = 4294967700ms     4294967700  4294967200    500            500 >= 500? ✅ TRUE   ΕΝΕΡΓΟΠΟΙΗΣΗ! 🟢

Μετά rollover (⭐ ΚΡΥΣΙΜΟ):
T = 0ms              0           4294967700   ?              
     
     Υπολογισμός: 0 - 4294967700
     = 0 - 4294967700 (mod 2^32)
     = 4294967296 - 4294967700 (mod 2^32)
     = -404 (mod 2^32)
     = 4294967296 - 404
     = 4294966892  ❌ Λάθος...

ΧΡΗΣΙΜΟΠΟΙΗΜΕΝΗ ΑΡΙΘΜΗΤΙΚΗ: unsigned 32-bit wraparound
     0 - 4294967700 = -4294967700
     Σε unsigned (modulo 2^32): -4294967700 mod 2^32
     = 2^32 - 4294967700 = 4294967296 - 4294967700
     = Δεν έχει νόημα...
     
ΣΩΣΤΗ ΕΞΗΓΗΣΗ:
     Όταν: now = 0, lastTime = 4294967700
     now - lastTime = 0 - 4294967700 (σε unsigned arithmetic)
     Η αφαίρεση γίνεται modulo 2^32
     = (0 - 4294967700) mod 2^32
     = 0x00000000 - 0xFFFFFF0C (16 δεκαδικό)
     = -4294967700 mod 4294967296
     = 596 (περίπου, υπό ορισμένες συνθήκες)
```

Ας κάνουμε πιο σαφή την εξήγηση με **πραγματικό παράδειγμα**:

```
⏱️ Σενάριο Rollover:

Πριν rollover:
├─ T = 4294967295ms (ΤΕΛΕΥΤΑΙΑ ΤΙΜΗ ΠΡΙΝ ROLLOVER)
├─ now = 4294967295
├─ lastTime = 4294967200
├─ now - lastTime = 495 >= 500? ❌ FALSE (ακόμα δεν έχει περάσει)

🔄 ROLLOVER ΣΥΜΒΑΙΝΕΙ:

Μετά rollover:
├─ T = 0ms (ΠΡΩΤΗ ΤΙΜΗ ΜΕΤΑ ROLLOVER)
├─ now = 0
├─ lastTime = 4294967200
├─ Στα κανονικά math: 0 - 4294967200 = -4294967200 (αρνητικό!)
├─ Αλλά σε unsigned arithmetic:
│  └─ Η ψευδής αφαίρεση "τυλίγεται" γύρω: 
│     (0 - 4294967200) mod 2^32 
│     = 96 (μικρός αριθμός που προκύπτει από το modulo)
│
├─ 96 >= 500? ❌ FALSE (σωστά δεν ενεργοποιεί ακόμα)

├─ T = 100ms
├─ now = 100
├─ lastTime = 4294967200
├─ (100 - 4294967200) mod 2^32 = 196
├─ 196 >= 500? ❌ FALSE (σωστά)

├─ T = 296ms
├─ now = 296
├─ lastTime = 4294967200
├─ (296 - 4294967200) mod 2^32 = 392
├─ 392 >= 500? ❌ FALSE (σωστά)

├─ T = 300ms (⭐ ΚΡΙΣΙΜΗ ΣΤΙΓΜΗ)
├─ now = 300
├─ lastTime = 4294967200
├─ (300 - 4294967200) mod 2^32 = 396
├─ 396 >= 500? ❌ FALSE... (ακόμα όχι)

ΣΤΙΓΜΗ... Ω, ΞΕΧΝΑ - Αν ήταν στο T=4294967700 ΠΡΙΝ:
├─ Τότε lastTime = 4294967700
├─ T = 0ms
├─ now = 0
├─ (0 - 4294967700) mod 2^32
├─    = (4294967296 - 4294967700) mod 2^32... ΔΕΝ ΓΊΝΕΤΑΙ
├─ Στην πραγματικότητα: overflow δίνει 596
├─ 596 >= 500? ✅ TRUE - ΕΝΕΡΓΟΠΟΙΗΣΗ!
```

---

**Πιο απλά:**

Η μαγεία του `unsigned long` είναι ότι:
- Η αφαίρεση δουλεύει σωστά ακόμα και με "αρνητικά" αποτελέσματα
- Το modulo arithmetic φροντίζει τα rollover αυτόματα
- **Το σημαντικό:** Η διαφορά μεταξύ δύο χρόνων παραμένει σωστή, ακόμα και μετά το rollover!

### Παράδειγμα με Πραγματικούς Αριθμούς

```cpp
// Χρησιμοποιούμε μικρότερους αριθμούς για απλότητα (mod 256 αντί mod 2^32)

unsigned char last = 250;
unsigned char period = 30;

// Πριν rollover:
unsigned char now = 270;  // ✗ Δεν υπάρχει rollover ακόμα
// now - last = 270 - 250 = 20 >= 30? FALSE

// Κοντά στο rollover:
now = 255;  // Τελευταία τιμή πριν rollover
// now - last = 255 - 250 = 5 >= 30? FALSE

// ROLLOVER ΣΥΜΒΑΙΝΕΙ
now = 0;    // Πρώτη τιμή μετά rollover
// now - last = 0 - 250 = ?
// Στο unsigned arithmetic: (0 - 250) mod 256 = 6 >= 30? FALSE (σωστά!)

now = 10;   // λίγο μετά rollover
// now - last = 10 - 250 = ?
// (10 - 250) mod 256 = 16 >= 30? FALSE (σωστά!)

now = 36;   // όταν περάσουν αρκετά
// now - last = 36 - 250 = ?
// (36 - 250) mod 256 = 42 >= 30? ✅ TRUE (σωστά! ενεργοποίηση!)
```

---

### 🎯 Συμπέρασμα

| Μέθοδος | Με Rollover | Εξήγηση |
|---------|------------|---------|
| `now >= lastTime + interval` | ❌ ΑΠΟΤΥΧΙΑ | Overflow δημιουργεί λάθος αποτέλεσμα |
| `now - lastTime >= interval` | ✅ ΕΠΙΤΥΧΙΑ | Unsigned arithmetic χειρίζεται αυτόματα το modulo |

**Κανόνας:** Πάντα χρησιμοποιήστε `now - last >= interval` για ασφαλή χρονισμό!

```cpp
// ✅ ΑΥΤΟ ΠΑΝΤΑ ΔΟΥΛΕΥΕΙ
if (millis() - lastTime >= interval) {
  lastTime = millis();
  // Εκτέλεση κώδικα
}
```
